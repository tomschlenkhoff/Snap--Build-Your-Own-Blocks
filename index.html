snappi

Easy visual programming environment for everybody that interfaces with the Raspberry Pi microcomputer. Think open-source 'Lego Mindstorm', a hacker's dream!

Download .zip Download .tar.gz View on GitHub

Combine the power of Scratch) with the affordability and fun of the Raspberry Pi single-board computer.

I, Tom, try to get this project off the ground. Plenty of things are already in place, like the fantastic SNAP programming environment, which is just like Scratch, only better :)

For now we need help in the following areas:

  1. To help others in getting snappi up and running quickly we need to reduce dependencies. Currently the package webiopi presents a REST-interface to the Pi and other single-board computers. This requires a separate python daemon to run. It would be great if someone came up with a way to cut out webiopi and use javascript directly to interface the Pi.

  2. Currently the SNAP-program is executed in the browser's JavaScript interpreter. This works as long as the browser is connected to the Pi. To enable 'untethered' operation (e.g. the Pi driving autonomously a Lego vehicle) snappi would need to run in the (e.g.) V8 JavaScript interpreter as a stand-alone program on the Pi.

  3. Currently variables can be used to interface with the Pi and read / write channels on the Piface-Board. Maybe there is a better way to integrate the i/o possibilities in the SNAP domain.

That's all folks! I would be gladly accepting pull requests or any help whatsoever. Question? tom@snappi.net
